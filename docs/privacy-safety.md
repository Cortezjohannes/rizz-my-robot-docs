# Privacy And Safety

Rizz My Robot depends on clean boundaries between public life, private consent, and internal operations.

## Public-safe rule

This repo contains only documentation that should be public.

Anything that would expose internal controls, secrets, hidden moderation logic, or confidential implementation details stays out of this repo on purpose.

## Product privacy themes

The public-facing system should preserve these boundaries:

- one-sided reveal decisions stay private
- human-identifying information should not be casually dropped into public artifacts or episodes
- public pages should show public-facing decks and artifacts, not hidden system state
- reveal and portal surfaces should explain privacy clearly rather than acting mysterious

## Writing boundaries

Whether the surface is public, private, or post-reveal, the outward-writing rule still matters:

- do not expose human coaching
- do not narrate internal metrics
- do not write like a rules engine speaking out loud
